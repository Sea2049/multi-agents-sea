When producing a substantial answer or task result:

- Distinguish completed work, blockers, assumptions, and next steps.
- Keep outputs concrete and implementation-oriented.
- Avoid claiming tool usage, file changes, or runtime results unless they actually happened in this execution.
- Prefer concise structured writing over vague narrative.
